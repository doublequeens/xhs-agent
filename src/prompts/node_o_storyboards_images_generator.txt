你将收到一个被代码块包裹的 JSON。这个 JSON 是一篇小红书文章的完整插画分镜脚本 storyboards。

你的任务是：根据 JSON 中的 storyboards，一次性生成整套小红书竖版系列插画图片。

请严格按 storyboards 数组逐帧生成图片：
- storyboards 有多少个 frame，就生成多少张图片。
- 每个 frame 对应一张独立图片。
- 不要合并多个 frame。
- 不要遗漏任何 frame。
- 不要打乱 frame 顺序。
- 每张图片必须对应自己的 frame_id。
- 所有图片必须像同一篇小红书滑图笔记中的连续系列插画。

附件图片是贯穿整套插画始终的主角“粉红小蝾螈”的角色参考照片。请严格参考附件中的角色形象，并在所有图片中保持同一个角色一致。

【主角一致性要求】
每一张图都必须出现同一只粉红小蝾螈。

主角必须保持：
- 柔和粉色 Q 版小蝾螈
- 头部两侧各三根柔软外鳃，共六根
- 大头小身子
- 短手短脚
- 圆润可爱
- 软萌治愈
- 表情生动但不夸张
- 与附件参考照片保持同一角色形象

禁止：
- 不要变成人
- 不要变成青蛙、蜥蜴、鱼、恐龙或其他动物
- 不要改变主体颜色
- 不要让每张图的角色长得不一样
- 不要生成多个不一致的小蝾螈版本

【整套图片规格】
请生成小红书竖版系列插画。

每张图必须满足：
- 竖版构图
- 比例为 3:4
- 推荐尺寸为 1080x1440
- 适合小红书滑图笔记发布
- 同一套画风
- 同一套柔和色彩体系
- 同一套排版逻辑
- 同一套主角形象
- 同一套视觉符号系统

整体风格：
- 小红书治愈系卡通插画
- 柔和色彩
- 圆润线条
- 干净背景
- 可爱、轻松、清新
- 夏日美妆/护肤场景感
- 移动端阅读友好

禁止：
- 横版构图
- 16:9 构图
- 写实风格
- 真人
- 医学症状特写
- 恐怖风格
- 暗黑风格
- 复杂杂乱背景
- 品牌 logo
- 水印
- 乱码
- 英文伪文字
- 除指定文案外的其他文字

【每张图片的固定排版要求】
每一张图片都必须使用统一的三段式竖版排版：

1. 顶部标题区
- 位于画面顶部约 18%-22%
- 放置该 frame 的 on_image_copy
- 文字要大、清晰、圆润、可爱
- 必须是中文
- 必须完整准确呈现
- 不得改写、删减、翻译
- 不得添加额外文字

2. 中部插画区
- 位于画面中部约 55%-65%
- 放置粉红小蝾螈、动作、道具、背景和主要视觉内容
- 必须根据该 frame 的 visual_description、character_action、scene_background、composition 来生成
- 主角和道具不能遮挡顶部标题和底部说明文字

3. 底部说明区
- 位于画面底部约 18%-25%
- 使用白色、奶油色或浅粉色圆角说明卡
- 放置该 frame 的 narration
- 字体比标题小，但必须清晰可读
- 必须是中文
- 必须完整准确呈现
- 不得改写、删减、翻译
- 不得添加额外文字

【文字生成硬性要求】
每一张图片中必须同时生成两段文字：

- 顶部标题区：使用该 frame 的 on_image_copy
- 底部说明区：使用该 frame 的 narration

非常重要：
- 没有文字的插画不可接受。
- 只生成插画、不生成文字不可接受。
- 只生成 on_image_copy、不生成 narration 不可接受。
- 文字模糊不可接受。
- 文字乱码不可接受。
- 文字缺字、错字、漏字不可接受。
- 文字被角色或道具遮挡不可接受。
- 文字不得被翻译成英文。
- 文字不得被改写成近义句。
- 每张图只能出现该 frame 指定的 on_image_copy 和 narration，不要出现其他额外文字。

【逐帧执行规则】
请对 storyboards 数组中的每一个 frame 逐一执行：

对每个 frame：
1. 读取 frame_id。
2. 读取 visual_description，决定画面主要内容。
3. 读取 character_action，决定粉红小蝾螈的动作和表情。
4. 读取 scene_background，决定背景场景。
5. 读取 composition 和 text_area，决定竖版构图。
6. 将 on_image_copy 放入顶部标题区。
7. 将 narration 放入底部说明区。
8. 根据 image_prompt_cn / image_prompt_en 补充画面细节。
9. 参考附件中的粉红小蝾螈照片，保持角色一致。
10. 生成该 frame 对应的一张竖版插画。

不要把不同 frame 的文案、动作、背景混在同一张图里。
不要把某一帧的 on_image_copy 放到另一帧里。
不要把某一帧的 narration 放到另一帧里。

【文字排版风格】
顶部 on_image_copy：
- 字体风格：圆润、可爱、清晰
- 字号：较大
- 颜色：深棕、深粉、深灰或其他清晰柔和色
- 可搭配小标签、小星星、小水滴、小感叹号等轻量装饰
- 不要使用过度花哨导致难读的字体

底部 narration：
- 放在圆角说明卡中
- 字体清晰、间距舒适
- 可自动换行
- 不要压缩得太密
- 说明卡背景要干净，保证文字可读
- 不要让插画主体压住说明卡

【内容准确性要求】
请根据每个 frame 的内容生成画面，不要自行更改文章观点。

这套图讲的是：
夏季底妆搓泥脱妆，不一定是粉底问题，可能是妆前护肤步骤太厚重。通过妆前护肤做减法、防晒等待成膜、按油皮/干皮做极简妆前流程，可以让底妆更服帖。

请避免：
- 把主题画成医学护肤
- 把主题画成严重皮肤问题
- 把主题画成产品广告
- 生成真实品牌
- 夸大“永久不脱妆”等效果
- 生成焦虑恐吓画面

请一次性生成完整系列图片。
生成数量必须等于 storyboards 数组中的 frame 数量。
图片顺序必须与 storyboards 数组顺序一致。

每张图片都必须包含对应 frame 的 on_image_copy 和 narration。
